Question 1: What is the DOM, and why is it important in web development?

The DOM (Document Object Model) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects, allowing developers to access, modify, and manipulate the document's content, structure, and styles dynamically using programming languages like JavaScript.

What is Event Bubbling?

Event bubbling is a concept in the DOM event model where an event triggered on a child element first executes its event handler and then propagates (or "bubbles up") to its ancestors, triggering their respective event handlers along the way.

When an event occurs on an element:

  • The event is first handled by the target element (the element where the event originated).
  • Theevent then propagates upward through its parent, grandparent, and other ancestor elements in the DOM tree, triggering any event handlers attached to them.

Why is Event Bubbling Useful?
  • Efficiency: Instead of adding event listeners to multiple child elements, you can add a single listener to a parent element and manage all child interactions using event bubbling.
  • Dynamic Elements: It helps handle events for elements added to the DOM dynamically after the page loads.
What is Event Delegation?

Event delegation is a programming technique in the DOM event model where a single event listener is attached to a parent element to handle events for its child elements. Instead of attaching individual event listeners to each child, the parent element's listener detects and responds to events from its children using the bubbling phase.
